# Plano de Teste Completo - Site Razor Industria

## Contexto
O projeto é um site vanilla HTML/CSS/JavaScript com backend Express.js para a empresa "Razor Industria". Não existe nenhum teste automatizado. O usuário quer verificar se 100% do site está funcionando.

**Stack:** Node.js/Express (server.js), HTML5, CSS3 custom, JavaScript vanilla, JSON flat-file database, IndexedDB client-side

**Páginas:** index.html, login.html, funcionario.html, permissoes.html, Funcionario/index.html (standalone)

---

## Fase 1: Verificação do Servidor

1. Instalar dependências (`npm install` se necessário)
2. Iniciar o servidor (`node server.js`) e verificar que escuta na porta 3000
3. Verificar que todos os arquivos estáticos são servidos corretamente

**Arquivo:** `server.js`

---

## Fase 2: Teste de Todas as Páginas HTML (Carregamento)

Verificar que cada página carrega sem erro de console:

| Página | Arquivo | O que verificar |
|--------|---------|-----------------|
| Landing Page | `public/index.html` | Hero, produtos, sobre, cotação, contato, footer, nav, WhatsApp button |
| Login | `public/login.html` | Formulário de login, link "Voltar ao site" |
| Painel Colaborador | `public/funcionario.html` | Sidebar, 7 abas (produção, conversor, mureta, malha, fotos, CRM, cancelamentos) |
| Permissões | `public/permissoes.html` | Tabela de usuários, formulário CRUD |
| Standalone | `Funcionario/index.html` | Calculadoras offline |

**Como testar:** Abrir cada página via browser/fetch e verificar HTTP 200 + ausência de erros de referência (CSS/JS 404)

## Fase 3: Teste de Todas as Rotas da API

### 3.1 Autenticação
- `POST /api/login` com credenciais válidas → 200 + token
- `POST /api/login` com credenciais inválidas → 401
- `POST /api/logout` com token → 200
- `GET /api/me` com token válido → 200 + dados do usuário
- `GET /api/me` sem token → 401

### 3.2 CRUD de Usuários (admin)
- `GET /api/usuarios` → 200 + lista
- `POST /api/usuarios` → criar usuário
- `PUT /api/usuarios/:id` → atualizar
- `DELETE /api/usuarios/:id` → remover
- Testar com token não-admin → 403

### 3.3 Vendedores
- `GET /api/vendedores` → 200 + lista de vendedores ativos

### 3.4 Categorias de Imagens
- `GET /api/categories` → 200 + lista
- `POST /api/categories` → criar categoria
- `DELETE /api/categories/:name` → remover

### 3.5 Imagens
- `GET /api/images/:category` → 200 + lista
- `GET /api/image/:cat/:file` → servir imagem
- `POST /api/upload/:category` → upload
- `DELETE /api/image/:cat/:file` → remover

### 3.6 Fotos de Clientes
- `GET /api/fotos/pastas` → 200 + lista
- `POST /api/fotos/pastas` → criar pasta
- `DELETE /api/fotos/pastas/:nome` → remover
- `GET /api/fotos/listar` → 200 + lista
- `POST /api/fotos/upload` → upload
- `GET /api/fotos/visualizar/:pasta/:arquivo` → servir foto
- `DELETE /api/fotos/:pasta/:arquivo` → remover
- `POST /api/fotos/mover` → mover foto

### 3.7 Tags
- `GET /api/tags/all` → 200 + lista
- `POST /api/tags` → adicionar tag
- `DELETE /api/tags/:id` → remover tag
- `GET /api/search` → buscar

### 3.8 Cancelamentos
- `GET /api/cancelamentos` → 200 + lista
- `POST /api/cancelamentos` → criar
- `PUT /api/cancelamentos/:id` → atualizar
- `DELETE /api/cancelamentos/:id` → remover

## Fase 5: Teste de Funcionalidade JavaScript (Painel - funcionario.js)

### 5.1 Navegação entre abas
- trocarTela() muda entre 7 seções
- Sidebar toggle funciona (desktop e mobile)
- Estado da sidebar persiste no localStorage

### 5.2 Calculadora de Produção
- Cálculo correto: Horas = M2Total / Producao
- Dias = ceil(Horas / 8)
- Previsão = ceil(Dias * 1.12) + diasFila
- Validação de inputs (comprimento > 0, altura > 0, quantidade > 0)
- Tabela editável com senhas

### 5.3 Conversor de Unidades
- Conversões corretas: mm ↔ cm ↔ m ↔ km
- Atualização em tempo real

### 5.4 Calculadora de Mureta
- Cálculo de materiais (blocos, meio-blocos, canaletas, cimento, areia, brita)
- Validação (metros > 0, fiadas >= 0)

### 5.5 Calculadora de Malha/Bobina
- Cálculo de peso por bobina e total
- Tabela de pesos configurável
- Validação (malha/bwg selecionados, metros > 0)

### 5.6 Gestão de Fotos
- Upload de fotos
- Criação/remoção de pastas
- Move entre pastas
- Visualização de fotos
- Exclusão com confirmação

### 5.7 CRM (IndexedDB)
- CRUD de clientes no IndexedDB
- Busca por nome/empresa
- Filtro por vendedor
- Export/Import JSON

### 5.8 Cancelamentos
- Criar solicitação de cancelamento
- Campo condicional "orçamento substituto" (motivo #6)
- Editar com senha
- Excluir com senha

## Método de Execução

Como não há framework de testes, o teste será feito via:
1. **curl/PowerShell** para testar API endpoints
2. **Node.js script** para testes automatizados de API (sem dependências externas)
3. **Verificação manual** de carregamento de páginas

### Script de Teste
Criar um arquivo `test-site.js` que:
1. Inicia o servidor
2. Testa todas as rotas HTTP via fetch
3. Verifica respostas HTTP corretas
4. Testa cenários de autenticação
5. Gera relatório de resultados
