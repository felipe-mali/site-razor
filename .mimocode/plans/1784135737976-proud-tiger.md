# Plano: Aba "Cancelamento de Orçamentos" — Apenas Comercial

## Contexto

O usuário quer adicionar uma nova aba no painel de Colaboradores (`funcionario.html`) exclusiva para o time Comercial (cargo `vendedor`). Essa aba será um formulário de **Solicitação de Cancelamento de Orçamento** com um template específico, protegido por senha para edição, e com persistência server-side para auditoria.

## Arquivos a modificar

| Arquivo | Ação |
|---------|------|
| `public/funcionario.html` | Adicionar botão da aba + seção HTML do formulário |
| `public/js/funcionario.js` | Adicionar lógica: acesso, senha, CRUD, renderização |
| `public/css/funcionario.css` | Estilos do formulário de cancelamento |
| `server.js` | Novos endpoints API para salvar/listar cancelamentos |
| `data/cancelamentos.json` | Novo arquivo de persistência (criado pelo server) |

## 1. HTML — `public/funcionario.html`

### 1.1 Botão de aba (após botão CRM, linha ~68)
```html
<button class="btn-menu" id="menu-cancelamento" onclick="trocarTela('cancelamento')" style="display:none">
  <span class="menu-icon">❌</span>
  <span class="menu-label">Cancelamentos</span>
</button>
```
- `display:none` por padrão — só aparece para vendedores (via JS)

### 1.2 Seção da aba (após seção CRM, antes de `</main>`)
Nova `<section id="tela-cancelamento" class="tela">` com:
- **Título:** "Solicitação de Cancelamento de Orçamento"
- **Card 1 — Formulário:** Campos conforme o template do usuário:
  - Solicitante (auto-preenchido com nome do usuário logado)
  - Data (auto-preenchida com data/hora atual)
  - Canal de contato (radio buttons: Whatsapp / Telefone PABX)
  - Matriz de ação por motivo (6 radio buttons, conforme modelo)
  - Campo condicional: "Qual orçamento substituiu" (aparece quando opção 6 selecionada)
  - Observações (textarea)
  - Botões: "Novo" (limpar) e "Enviar Solicitação"
- **Card 2 — Histórico:** Tabela com solicitações anteriores (carregada do servidor)
  - Colunas: Data, Solicitante, Motivo, Status
  - Botão "Excluir" para cada registro (apenas com senha)
- **Área de bloqueio:** Quando o formulário está bloqueado, exibe overlay com mensagem "Digite a senha para editar" + campo de senha + botão "Desbloquear"

### 1.3 Template de impressão (formato exato pedido pelo usuário)
Quando o usuário clicar "Enviar", o sistema gera o texto formatado:
```
SOLICITAÇÃO DE CANCELAMENTO DE ORÇAMENTO
SOLICITANTE: [nome] - Razor Comercial  |  DATA: [data]

QUAL CANAL DE CONTATO ONDE FOI INFORMANDO O CANCELAMENTO
(x) Whatsapp	(   ) Telefone PABX

MATRIZ DE AÇÃO POR MOTIVO:
(x) 1. Preço (não aprovou o valor)
(   ) 2. Concorrente — foi para outro fornecedor
...

OBSERVAÇÕES
[texto]
```

## 2. JavaScript — `public/js/funcionario.js`

### 2.1 Constantes
```javascript
const SENHA_CANCELAMENTO = "Lim@020707";
```

### 2.2 Controle de acesso (no DOMContentLoaded)
```javascript
// Mostrar aba de cancelamento apenas para vendedores
if (user?.cargo === 'vendedor') {
  document.getElementById('menu-cancelamento').style.display = '';
}
// Se a aba salva era cancelamento e user não é vendedor, ir para producao
if (telaSalva === 'cancelamento' && user?.cargo !== 'vendedor') {
  trocarTela('producao');
}
```

### 2.3 Funções da aba
- `cancelamentoDesbloquear()` — Valida senha `Lim@020707`, desbloqueia formulário
- `cancelamentoBloquear()` — Bloqueia formulário novamente
- `cancelamentoEnviar()` — Valida campos obrigatórios, monta texto formatado, envia para API
- `cancelamentoLimpar()` — Reseta formulário
- `cancelamentoCarregar()` — Busca histórico do servidor e renderiza tabela
- `cancelamentoExcluir(id)` — Remove registro (com confirmação)
- `cancelamentoMontarTexto(dados)` — Gera o texto no formato exato do template

### 2.4 Validações
- Campos obrigatórios: canal de contato, motivo (pelo menos 1), observações
- Se motivo 6 selecionado: campo "qual orçamento substituiu" é obrigatório
- **Senha NÃO necessária para criar** — qualquer vendedor pode criar livremente
- **Senha necessária para EDITAR e EXCLUIR** registros existentes

## 4. Server — `server.js`

### 4.1 Arquivo de persistência
`data/cancelamentos.json` — Array de objetos:
```json
[
  {
    "id": "unique-id",
    "solicitante": "André",
    "nomeExibicao": "Comercial 2",
    "data": "15/07/2026 10:29",
    "canalContato": "whatsapp",
    "motivo": 1,
    "orcamentoSubstituiu": null,
    "observacoes": "...",
    "textoFormatado": "...",
    "dataCriacao": "2026-07-15T10:29:00.000Z"
  }
]
```

### 4.2 Endpoints
```
GET    /api/cancelamentos          — Lista todos (autenticado + cargo vendedor)
POST   /api/cancelamentos          — Cria novo (autenticado + cargo vendedor, sem senha)
DELETE /api/cancelamentos/:id      — Exclui (autenticado + cargo vendedor)
```
- POST: qualquer vendedor pode criar
- DELETE: qualquer vendedor pode excluir (senha é validada no client-side, mesma lógica do projeto existente)

Middleware `requireVendedor`:
```javascript
function requireVendedor(req, res, next) {
  if (req.user?.cargo !== 'vendedor') {
    return res.status(403).json({ error: 'Acesso restrito ao time Comercial' });
  }
  next();
}
```
