# Plano: Corrigir Login/Autenticação - Razor Indústria

## Diagnóstico

O projeto é um servidor **Node.js/Express** (não ASP.NET). O erro de login/autenticação tem **4 causas raiz**:

### Problema 1: Estrutura de permissões inconsistente (CRÍTICO)

`data/usuarios.json` guarda permissões como **propriedades flat** no objeto do usuário:
```json
{ "pode_ver_funcionario": true, "pode_editar_imagens": false, "pode_gerenciar_permissoes": true }
```

Mas `POST /api/usuarios` (server.js:91) cria usuários com permissões **aninhadas** em objeto `permissoes`:
```js
users[usuario] = { nome, senha, cargo, ativo, permissoes: permissoes || {} };
```

Resultado: usuários criados via aba "Permissões" ficam sem acesso admin porque `req.user.pode_gerenciar_permissoes` é `undefined` (está em `permissoes.pode_gerenciar_permissoes`).

### Problema 2: PUT /api/usuarios destrói permissões flat (CRÍTICO)

`PUT /api/usuarios/:id` (server.js:106-113) usa spread que sobrescreve propriedades flat existentes por um objeto nested `permissoes`, destruindo as permissões.

### Problema 3: Login retorna senha no payload (SEGURANÇA)

`server.js:60` espalha TODOS os campos do usuário no response, incluindo `senha`:
```js
res.json({ success: true, token, user: { nome: users[usuario].nome, ...users[usuario] } });
```

A senha fica visível no `localStorage` do navegador.

### Problema 4: DELETE /api/usuarios/:id não existe (BUG)

`permissoes.js:104` chama `DELETE /api/usuarios/${id}` mas essa rota **não existe** no server.js. Botão "Excluir" não funciona.

## Plano de Correção

### Arquivos a modificar:
- `server.js` (backend)
- `public/js/permissoes.js` (frontend de gerenciamento)

### Alteração 1: Corrigir `POST /api/usuarios` (server.js:88-100)
Salvar permissões como **propriedades flat**, não como objeto nested:
```js
// ANTES:
users[usuario] = { nome, senha, cargo, ativo, permissoes: permissoes || {} };

// DEPOIS:
users[usuario] = {
  nome, senha, cargo,
  ativo: ativo !== false,
  pode_ver_funcionario: permissoes?.pode_ver_funcionario || false,
  pode_ver_imagens: permissoes?.pode_ver_imagens || false,
  pode_editar_imagens: permissoes?.pode_editar_imagens || false,
  pode_gerenciar_permissoes: permissoes?.pode_gerenciar_permissoes || false
};
```

### Alteração 2: Corrigir `PUT /api/usuarios/:id` (server.js:102-119)
Atualizar permissões como flat props e não sobrescrever com nested:
```js
// ANTES:
...(permissoes && { permissoes })

// DEPOIS:
...(permissoes && {
  pode_ver_funcionario: permissoes.pode_ver_funcionario ?? false,
  pode_ver_imagens: permissoes.pode_ver_imagens ?? false,
  pode_editar_imagens: permissoes.pode_editar_imagens ?? false,
  pode_gerenciar_permissoes: permissoes.pode_gerenciar_permissoes ?? false
})
```

### Alteração 3: Remover senha do payload do login (server.js:60)
```js
// ANTES:
res.json({ success: true, token, user: { nome: users[usuario].nome, ...users[usuario] } });

// DEPOIS:
const { senha: _, ...userData } = users[usuario];
res.json({ success: true, token, user: { ...userData, usuario } });
```

### Alteração 4: Adicionar rota DELETE faltante (server.js, após linha 119)
```js
app.delete('/api/usuarios/:id', authenticate, requireAdmin, (req, res) => {
  const users = JSON.parse(fs.readFileSync('data/usuarios.json', 'utf8'));
  if (users[req.params.id]) {
    delete users[req.params.id];
    fs.writeFileSync('data/usuarios.json', JSON.stringify(users, null, 2));
    res.json({ success: true });
  } else {
    res.status(404).json({ error: 'Usuário não encontrado' });
  }
});
```

### Alteração 5: Corrigir frontend permissoes.js para enviar flat props
`permissoes.js:126-131` já envia `permissoes: { ... }` — isso é correto, o backend é que precisa mapear para flat. Não precisa mudar o frontend.
