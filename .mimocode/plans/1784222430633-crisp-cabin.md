# Plano: Correção de Encoding + Nova Aba Dashboard Power BI

## Resumo
Dois problemas a resolver:
1. **Caracteres aleatórios** no CRM e Fotos (causado por encoding UTF-8 double-encoded)
2. **Nova aba "Dashboard"** na sidebar do vendedor com iframe do Power BI

---

## Problema 1: Correção de Encoding UTF-8

### Causa Raiz
Os arquivos `funcionario.js` e `server.js` foram salvos em UTF-8, mas depois re-lidos/re-escritos como Latin-1/Windows-1252, corrompendo todos os caracteres acentuados, emojis e símbolos especiais.

### Arquivos Afetados
- `public/js/funcionario.js` - Caracteres visíveis na UI (emojis quebrados, placeholders, mensagens)
- `server.js` - Mensagens de erro da API retornam caracteres quebrados ao navegador

### Solução
Executar o script `fix-enc.js` que já existe no projeto:

```bash
node fix-enc.js public/js/funcionario.js
node fix-enc.js server.js
```

### Impacto Esperado
- **CRM**: Ícone do estado vazio (👤) vai aparecer corretamente
- **Fotos**: Ícones de pastas (📁) e câmera (📷) vão funcionar; placeholders de produção (m²/h) vão ficar corretos
- **Server**: Mensagens de erro em português vão ser legíveis

### Verificação
1. Abrir o site no navegador
2. Navegar até CRM - verificar que o ícone 👤 aparece quando não há clientes
3. Navegar até Fotos - verificar que os ícones 📁 aparecem nas pastas
4. Testar criação de pasta - confirmar que acentos são preservados (regex no server.js linha 221)

## Problema 2: Nova Aba "Dashboard" com Power BI

### Estrutura Atual
- Sidebar: `public/funcionario.html` (linhas 34-96)
- Tab switching: `public/js/funcionario.js` (função `trocarTela()` linha 107)
- Grupos da sidebar: `calculadoras`, `comercial`, `producao`, `logistica`
- Vendedor tem acesso aos grupos: `calculadoras` e `comercial`

### Implementação

#### 1. Adicionar botão na sidebar (funcionario.html)
- **Localização**: Grupo "comercial" (após o botão de Cancelamentos, linha ~73)
- **Código**: Botão com `onclick="trocarTela('dashboard')"` e ícone de gráfico

#### 2. Criar seção da aba (funcionario.html)
- **Localização**: Após a seção de cancelamentos (linha ~248)
- **Conteúdo**: 
  - `<section id="tela-dashboard" class="tela">`
  - Container com iframe responsivo para Power BI
  - URL placeholder que será substituída pelo usuário

#### 3. Atualizar JavaScript (funcionario.js)
- **Adicionar no mapa `trocarTela()`** (linha ~107):
  ```js
  dashboard: 'tela-dashboard'
  ```
- **Adicionar na lista de telas permitidas para vendedor** (linha ~48):
  ```js
  var telasVendedor = ['crm','cancelamentos','fotos','dashboard','producao','conversor','mureta','malha'];
  ```

#### 4. Estilização (funcionario.css)
- Estilo para o container do iframe do Power BI
- Responsividade para diferentes tamanhos de tela

### Arquivos a Modificar
