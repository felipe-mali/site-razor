# Plano: Página de Cancelamento de Orçamentos

## Resumo
Adicionar uma nova aba "Cancelamentos" no painel do colaborador (`funcionario.html`) onde vendedores podem criar solicitações de cancelamento de orçamento. A edição/exclusão dessas solicitações requer a senha `Lim@020707`. Os dados são salvos no servidor via API REST (JSON).

---

## Arquivos a Criar/Modificar

### 1. `public/js/cancelamentos.js` (NOVO)
JavaScript dedicado à aba de cancelamentos. Funções principais:
- **Verificar permissão**: apenas `cargo === 'vendedor'` pode criar
- **`cancelamentoNovoUI()`**: abre modal/formulário de criação com o formato padrão:
  - Título: "SOLICITAÇÃO DE CANCELAMENTO DE ORÇAMENTO"
  - CAMPO: Solicitante (auto-preenchido com nome do usuário logado + " - Razor Comercial")
  - CAMPO: Data (auto-preenchido com data/hora atual)
  - CANAL DE CONTATO: Radio buttons (WhatsApp / Telefone PABX)
  - MATRIZ DE AÇÃO POR MOTIVO: 6 opções radio, sendo que a opção 6 exige campo obrigatório de orçamento substituto
  - CAMPO: Observações (textarea)
- **`cancelamentoSalvar()`**: salva via POST `/api/cancelamentos`
- **`cancelamentoEditarUI(id)`**: solicita senha `Lim@020707` antes de abrir modal de edição
- **`cancelamentoExcluir(id)`**: solicita senha `Lim@020707` antes de confirmar exclusão
- **`cancelamentosRenderizar()`**: lista todas as solicitações em cards formatados

## Fluxo de Uso

### Criar cancelamento (vendedor)
1. Vendedor clica na aba "Cancelamentos"
2. Clica em "+ Nova Solicitação"
3. Formulário abre com nome e data auto-preenchidos
4. Seleciona canal, motivo e preenche observações
5. Clica "Salvar" → grava no servidor

### Editar/Excluir (requer senha)
1. Ao clicar "Editar" ou "Excluir", abre modal pedindo senha
2. Senha correta: permite a ação
3. Senha incorreta: erro e bloqueio
